# MultiLane - Game Development Framework\n\n## Overview\n\n**MultiLane** is a comprehensive game development framework built on Unity, combining:\n- **Physics-based character movement** with thruster-based propulsion\n- **Destructible voxel-based worlds** with advanced material systems\n- **Modular architecture** designed for scalability and extensibility\n\n## Quick Start\n\n### Prerequisites\n- Unity 2022 LTS or newer\n- Git\n- Python 3.8+ (for utility scripts)\n\n### Setup\n\n```bash\n# Clone the repository\ngit clone https://github.com/XhakaTech/MultiLane.git\ncd MultiLane\n\n# Open in Unity\n# File > Open Project > Select this folder\n```\n\n### First Run\n\n1. Open the project in Unity\n2. Navigate to the Scene folder and open the Main scene\n3. Press Play to test the current implementation\n\n## Project Structure\n\n```\nMultiLane/\n├── Assets/\n│   ├── Scripts/                 # Main gameplay code\n│   │   ├── Core/               # Core systems (movement, physics)\n│   │   ├── Movement/           # Physics and thruster systems\n│   │   ├── Visuals/            # Visual controllers\n│   │   ├── Input/              # Input handling\n│   │   ├── Data/               # ScriptableObject configurations\n│   │   └── World/              # Voxel world systems\n│   ├── Scenes/                 # Game scenes\n│   ├── Resources/              # Localization, configs\n│   ├── Settings/               # Project settings assets\n│   └── ...\n├── Docs/                        # Documentation\n│   ├── Arquitectura_Esfera.md  # Character system architecture\n│   ├── GameDesign_VoxelWorld.md # World design document\n│   └── ...\n├── Tools/                       # Utility scripts\n├── ProjectSettings/             # Unity project configuration\n└── README.md                    # This file\n```\n\n## Core Systems\n\n### 1. Character Movement (Sphere Entity)\n\nA physics-based platformer character controlled by 4 directional thrusters (WASD):\n- **Fuel System**: Thrusters consume fuel, fuel regenerates over time\n- **Physics Integration**: Full Rigidbody2D physics with angular velocity\n- **Modular Components**: Upgradeable and swappable systems\n- **Visual Feedback**: Sprite rotation, overlay tilting, fuel bar display\n\n**Architecture:** See `Docs/Arquitectura_Esfera.md` for detailed design\n\n### 2. Voxel World\n\nA destructible tile-based world with:\n- **Material System**: 14+ materials with different destruction properties\n- **Damage Types**: Additive, Restrictive, and Custom damage modes\n- **Physics Debris**: Destroyed tiles become physics entities\n- **Performance**: ECS + Burst Compiler for handling 10k+ particles\n\n**Design Document:** See `Docs/GameDesign_VoxelWorld.md`\n\n### 3. Weapon System\n\nMultiple weapon types with different damage profiles:\n- **Ballistic**: Pistol (15 dmg), Rifle (25 dmg)\n- **Explosive**: RPG (80 dmg), Granada (60 dmg)\n- **Penetrating**: PerforadorA, PerforadorB (special behaviors)\n\n## Key Features\n\n✅ **Modular Architecture**: Systems communicate via events and dependency injection  \n✅ **Scalable Physics**: Handle thousands of physics objects efficiently  \n✅ **Extensible Design**: Easy to add upgrades, skills, and new features  \n✅ **Comprehensive Documentation**: Architecture diagrams and system references  \n✅ **Localization Ready**: Support for 9 languages (ES, FR, DE, ID, IT, PL, PT, TR, VI)  \n✅ **Editor Tools**: Automated setup and configuration tools  \n\n## Documentation\n\n- **[ARCHITECTURE.md](ARCHITECTURE.md)** - Create a test scene\n- **Window > Indoctrination > SetupTools > Material & Weapon Setup** - Create configs\n- **Window > Indoctrination > SetupTools > Sprite Generator** - Generate sprites\n\n## Extension Guide\n\n### Adding Upgrades\n\n```csharp\nvar upgrade = GetUpgradeConfig();\nsphereEntity.ApplyUpgrade(\"fuel_boost\", upgrade);\n```\n\n### Adding Skills\n\nCreate a new module in `Scripts/Abilities/` and extend `SphereEntity`:\n```csharp\npublic void ActivateDashSkill(Vector2 direction) { ... }\n```\n\n### Adding AI Enemies\n\nReplace `PlayerInputHandler` with `AIInputHandler`:\n```csharp\nvar aiInput = gameObject.AddComponent<AIInputHandler>();\naiInput.Initialize(thrusterController);\n```\n\n## Performance Considerations\n\n- **Debris System**: Uses ECS + Burst Compiler to handle 10k+ particles\n- **Physics**: Chunks processed in background using NativeArrays\n- **Memory**: Tilemap data stored in cache-friendly NativeArray structure\n- **Rendering**: Sprite batching and layer-based sorting optimization\n\n## Known Limitations\n\n- Building destruction system is in v2 (simplified from original plan)\n- Interior reveal system is basic (wall toggle on player entry)\n- Stress structural system removed (can be re-added in future versions)\n\n## Contributing\n\n1.
